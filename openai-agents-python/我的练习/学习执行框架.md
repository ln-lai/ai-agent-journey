## 每个练习的固定框架

每个练习都按 5 步做：

1. 写一句话目标：这个 Agent 解决什么问题。
2. 画结构：用户输入 -> Agent -> 工具/子 Agent -> 最终输出。
3. 先跑通最小代码。
4. 加一个异常场景：工具失败、用户问偏、模型答错。
5. 写 150 到 300 字复盘。

## 简历项目建议

我建议你优先做：个人研究 / 内容情报 Agent。

原因：

- 你本来就在做 X/Twitter、热点、内容分析相关事情。
- 项目能体现真实需求，不像玩具 demo。
- 可以自然用上工具、路由、并行、judge、记忆。

第一版项目范围：

- 输入一个主题。
- 搜索或读取候选内容。
- 多 Agent 分析：价值判断、反方观点、写作角度。
- judge Agent 选出最值得写的一条。
- 输出一篇结构化日报或 thread 草稿。

面试讲法：

```text
这个项目不是简单聊天机器人，而是一个内容研究 workflow。
我用 routing 判断任务类型，用 tools 获取外部信息，用 parallelization 生成多个候选判断，再用 judge Agent 做质量评估，最后把结果结构化输出。
```
